# Project Specifications: Modular Signal Generator

This document outlines the architecture, features, and components of the Python Tkinter signal generator application.

## 1. Core Objective
The primary goal is to create a didactic, interactive application for generating complex, non-stationary signals. Users can build a signal by combining various base signal sources and then applying a sequence of anomalies, noise, and distortions. The tool is designed for visualizing how these components affect a signal in the time domain.

## 2. Project Architecture
The project is structured into multiple files and directories to ensure modularity and readability.

- **`main.py`**: Contains the main `SignalGeneratorApp` class, which manages the UI layout, component lists, plotting, and user interaction logic. It orchestrates the overall signal generation pipeline.
- **`utils.py`**: A utility file containing helper functions, notably `create_slider_entry`, which generates the standard UI for a parameter slider with a linked text box.
- **`components/`**: A directory for all signal processing blocks.
    - **`base.py`**: Defines the abstract `BaseComponent` class, which all other components inherit from. It establishes the core interface (`__init__`, `get_config_frame`, `generate`, `to_dict`, `from_dict`, etc.). It also defines `SignalComponent` and `AnomalyComponent` subclasses.
    - **`signals.py`**: Contains all classes that generate a primary signal (e.g., Sine, Chirp). These are additive.
    - **`anomalies.py`**: Contains all classes that modify an incoming signal (e.g., noise, jumps, drift). These are applied sequentially.
- **`tests/`**: A directory for unit tests.
    - Uses Python's built-in `unittest` framework.
    - `test_signals.py` and `test_anomalies.py` provide basic validation for each component.
- **`example_signal.json`**: A comprehensive, commented JSON file that serves as a reference and an example for the save/load functionality.

## 3. User Interface (UI) and Functionality
The UI is built with Tkinter and is divided into a left control panel and a right plot area.

### Control Panel:
- **Configuration**: "Save Config" and "Load Config" buttons for JSON-based state management.
- **Signal Sources**: A listbox to manage the additive base signals. Users can add/remove signals from a dropdown.
- **Anomalies & Noise**: A second listbox to manage the sequential anomalies applied after the base signals are summed.
- **Component Preview**: A small plot that displays ONLY the output of the currently selected component from either listbox. Its axes are synced with the main plot.
- **Parameters**: A dynamic frame that shows the configuration sliders and entry boxes for the currently selected component.

### Core Behavior:
- **Live Updates**: The plot updates in real-time as sliders are dragged.
- **Stable Text Entry**: Typing values into text boxes does not cause the UI to lose focus. The plot updates only when Enter is pressed or focus is lost.
- **Dynamic Time Sliders**: Sliders for time-based parameters (e.g., `impulse_time`) automatically adjust their range to match the visible duration on the main plot.
- **Visible Periods**: A slider below the main plot controls the number of periods of the highest-frequency signal to display, effectively controlling the time axis.
- **Clean Exit**: The application terminates correctly when the main window is closed.

## 4. Signal Generation Pipeline
1. The total duration is calculated based on the "Visible Periods" slider and the highest frequency among all active **Signal Sources**.
2. A time vector `t` is created.
3. An initial signal `y` of all zeros is created.
4. **Signal Stage**: Loop through all components in the "Signal Sources" list. For each component, add its output to `y`.
5. **Anomaly Stage**: Loop through all components in the "Anomalies & Noise" list. For each component, its `generate` method is called with the output of the previous stage, applying the effect sequentially.
6. The final `y` is plotted.

## 5. Component Reference

### Signal Sources (`signals.py`)
- **Sine**: `amplitude`, `frequency` (Hz), `phase` (degrees).
- **Cosine**: `amplitude`, `frequency` (Hz), `phase` (degrees).
- **Square**: `amplitude`, `frequency` (Hz), `phase` (degrees), `duty_cycle` (0.0-1.0).
- **Chirp**: `amplitude`, `start_freq` (Hz), `end_freq` (Hz).
- **Sine (Varying Freq)**: `amplitude`, `start_freq` (Hz), `end_freq` (Hz), `change_time` (s).

### Anomalies & Noise (`anomalies.py`)
- **Gaussian Noise**: `std_dev` (noise power), `seed` (integer for reproducibility). Has a "Regenerate" button to get a new random seed.
- **Amplitude Jump**: `jump_size`, `jump_time` (s). This is a step function applied to the incoming signal.
- **Impulse Noise**: `amplitude`, `impulse_time` (s).
- **Outlier**: `value` (the absolute value to set), `outlier_time` (s).
- **Bias**: `offset` (a constant DC value to add).
- **Drift**: `slope` (in units per second).
- **Signal Dropout**: `start_time` (s), `duration` (s).
- **Saturation**: `threshold` (clips the signal at +/- this value).
- **Time Delay**: `delay` (s).

## 6. Configuration Management
- The state of all active components can be saved to a JSON file.
- The `to_dict()` method on each component serializes its parameters.
- The `from_dict()` method deserializes and sets the component's state.
- The `main.py` orchestrates this process, clearing and rebuilding the component lists when a file is loaded.
